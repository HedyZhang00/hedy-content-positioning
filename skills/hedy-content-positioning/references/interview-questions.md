# 动态采访问题

## 通用规则

- 一次只问一个问题。
- 先简短复述上一条回答中新增的事实，再提出下一问。
- 默认采访 8 至 12 个核心问题；证据充分时可以少问，证据不足时可以追问。
- 不连续追问同一抽象概念；改问具体对象、场景、行动和结果。
- 不在采访完成前提前生成三条内容主线。
- 不必逐字问完问题库；优先补齐缺失的证据。

## 第一阶段：业务现实

目标：确定内容最终需要连接什么。

优先问题：

1. 你现在主要为别人解决什么问题？
2. 别人过去为什么愿意找你、信任你或付费？
3. 未来半年，你最希望内容帮助你实现什么？

若用户没有产品，改问：过去别人最常因为什么问题找你？未来半年你最想验证哪种服务？

## 第二阶段：优势证据

目标：从真实经历中识别能力，不接受只有形容词的答案。

优先问题：

1. 哪些问题别人会反复来问你？
2. 你做过哪些有明确结果的事情？
3. 哪些事你觉得自然，别人却觉得困难？
4. 选一个最能证明的案例：对方是谁、遇到什么问题、你做了什么、结果怎样？

动态追问：

- “我擅长沟通” → 最近一次帮助谁处理什么问题？你采取了什么行动？
- “客户反馈很好” → 对方具体说了什么？有没有行为或结果变化？
- “我没有案例” → 你为自己完成过什么实验？有什么可观察结果？

## 第三阶段：优先读者

目标：从“所有人都可以看”收窄到未来半年最优先的一类人。

优先问题：

1. 哪类人的问题你最理解？
2. 她现在处于什么阶段？
3. 她会用什么原话描述自己的卡点？
4. 你愿意长期和这类人打交道吗？为什么？

动态追问：

- “所有人” → 如果未来半年只能吸引一类最可能成为客户的人，你选谁？
- 多个人群 → 分别说出证据和商业连接，再选未来半年优先级。

## 第四阶段：长期素材

目标：判断是否真的有东西可持续表达。

优先问题：

1. 你已经积累了哪些经历、案例或观察？
2. 哪些话题你愿意持续讲一年？
3. 哪些话题虽然热门，但你缺少证据或不愿长期讲？

动态追问：

- 五个以上兴趣 → 分成业务证据、长期兴趣、短期好奇三组。
- 只说热门赛道 → 追问用户的独特经历、案例和反常识判断。
- 提供主页或文章 → 作为证据审阅，不默认现有内容就是正确定位。

## 第五阶段：候选校验

确认四类证据已经足够后：

1. 生成 2 至 3 个定位候选。
2. 对每个候选说明受众、业务连接、适配证据、风险、待验证假设和四维判断。
3. 只问一个选择问题：你想选择、组合还是否决哪个候选？
4. 用户选择后，再输出完整定位卡、三条主线和 7 天验证计划。

## 中途停止

用户不想继续时，只输出：

- 已确认事实；
- 当前可能方向；
- 仍缺少的关键证据；
- 下次继续时从哪个问题开始。

不得生成看似完整但缺少证据的终稿。
