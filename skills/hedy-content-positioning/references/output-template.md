## 1. 输出目标与原则

最终结果用于帮助用户作出定位与行动决策，不输出长篇分析报告。

默认输出两张独立知识卡片：

1. **内容定位卡**：回答“我服务谁、讲什么、为什么是我、内容连接什么业务”；
2. **7 天定位验证卡**：回答“接下来怎么测试、观察什么、如何作出取舍”。

最终结论统一称为“内容定位假设 V1.0”，不得写成命定方向、唯一答案或确定性人格结论。

### 表达规则

- 结论优先，每个模块控制在 1 至 4 行。
- 只保留决策需要的信息：人群、问题、证据、取舍、商业连接和行动。
- 显式区分已知事实、合理推断和待验证假设。
- 不编造用户没有提供的数据、案例、反馈或付费需求。
- 两张卡片总字数建议控制在 700 至 1200 字；图片版应进一步压缩。
- 除非用户明确要求，不在卡片后继续输出额外长篇解释。

## 2. 定位候选

用户确认候选前，先输出 2 至 3 个方向：

```markdown
### 候选 N｜名称

- 服务谁：
- 核心问题：
- 为什么是你：
- 商业连接：
- 持续素材：
- 主要风险：
- 待验证假设：
- 五维判断：证据强度【强/中/弱】｜商业连接【强/中/弱】｜长期能量【强/中/弱】｜差异化【强/中/弱】｜验证成本【低/中/高】
```

候选后只问：你想选择、组合还是否决哪个候选？用户确认前不得生成最终卡片。

## 3. 卡片一：内容定位卡

```markdown
CONTENT POSITIONING

# 一人公司内容定位卡
> 内容定位假设 V1.0

## 01｜你的定位
我帮助【一类优先人群】，解决【具体阶段的核心问题】，通过【独特方法或能力】，让她们能够【期望改变】。

## 02｜你服务谁
【具体身份】正处于【具体阶段】，最需要解决【具体问题】。

## 03｜唯一核心主线
围绕「【未来半年最值得反复讨论的中心问题】」持续输出。

你希望别人记住你是因为：【一句清晰记忆点】。

## 04｜为什么是你
- 【真实案例、结果或可验证经历 1】
- 【真实案例、结果或可验证经历 2】
- 【真实案例、结果或可验证经历 3】

## 05｜内容结构
- 核心主线：【中心问题】
- 证明线：【案例、方法、结果、失败复盘或数据】
- 人格与关系线：【价值观、经历、选择或相关观察】

证明线和人格与关系线都服务于核心主线，不是三个独立定位。

## 06｜商业连接假设
- 业务方向：【内容最可能连接的业务】
- 可能付费的人：【最可能产生付费行为的人群】
- 付费问题：【对方愿意花钱解决的具体问题】
- 最小产品或服务：【当前能力范围内可测试的轻量方案】
- 下一步行动：【私信、问卷、预约或社群中的一个】
- 已有商业证据：【事实；没有则写暂无直接证据】
- 最大待验证假设：【一句话】

## 07｜暂时不要做
- 不要同时讲【容易分散定位的方向】。
- 不要为了流量持续吸引【非目标人群】。
- 不要在证据不足时包装成【尚未成立的身份】。

## 08｜判断状态
- 当前置信度：【高/中/待验证】
- 已知事实：【1 至 3 条】
- 合理推断：【1 至 2 条】

> 总结 / SUMMARY
> 你真正需要放大的，不是【过去容易陷入的方向】，而是【最值得建立的核心认知】。未来半年，优先围绕「【核心主线】」积累内容、案例和反馈。
```

一句话定位控制在 35 至 60 字。只选择一类优先人群和一个内容中心。“为什么是你”只保留最强的三条真实证据。证据不足时明确写成公开实践型或成长型定位。

商业连接只搭桥，不展开完整产品矩阵、详细定价、成交话术、私域 SOP 或高客单交付方案。

## 4. 卡片二：7 天定位验证卡

```markdown
7-DAY VALIDATION

# 7 天定位验证卡
> 不继续猜定位，用真实内容验证定位。

## 01｜本轮要验证
【目标人群】是否会因为你讨论「【核心问题】」而产生共鸣、信任或进一步行动。

## 02｜7 天行动
- DAY 1｜身份与问题：【内容主题】｜验证：【问题定义】｜引导：【留言】
- DAY 2｜典型痛点：【内容主题】｜验证：【用户是否说这就是我】｜引导：【补充场景】
- DAY 3｜核心观点：【内容主题】｜验证：【观点记忆点】｜引导：【同意或反对】
- DAY 4｜方法框架：【内容主题】｜验证：【收藏、尝试或追问】｜引导：【实践反馈】
- DAY 5｜案例证据：【内容主题】｜验证：【信任是否增强】｜引导：【私信问题】
- DAY 6｜人格与选择：【内容主题】｜验证：【价值观认同】｜引导：【分享选择】
- DAY 7｜商业行动：【内容主题】｜验证：【是否愿意进入下一步】｜引导：【私信/问卷/预约】

## 03｜重点观察
- 目标用户信号：【具体评论、问题、自我识别和新增关注】
- 商业行动信号：【私信、问卷、预约、服务或合作询问】
- 个人能量信号：【最容易持续和最明显消耗的主题】

## 04｜7 天后判断
- 保留：目标用户明确回应，出现具体问题或下一步行动，同时你愿意继续表达。
- 调整：有流量但人群不对，或有共鸣却没有具体问题与商业行动。
- 放弃或降级：多次发布后没有目标用户回应，也缺少素材、意愿或业务连接。

## 05｜下一轮迭代
带回表现最好内容、评论原话、私信咨询、新增目标用户和个人能量记录，再生成“内容定位假设 V1.1”。

> 总结 / SUMMARY
> 定位不是一次想清楚的，而是在持续表达中逐步收敛的。7 天的目标不是证明你一定正确，而是减少一个错误方向。
```

7 天计划不能只是机械日更。每条内容都必须标明验证目的和行动引导。不得只用点赞量判断定位。

## 5. 信息不足时的输出

六项最小证据门槛缺少两项以上时，不得强行填满两张卡片。只输出：

```markdown
# 定位信息缺口卡

- 已确认事实：
- 当前无法确定：
- 缺失的关键证据：
- 下一步只补充的一项信息：

> 目前还不足以形成可靠的内容定位假设。不是因为你没有定位，而是缺少【具体证据】。
```

然后继续一次只问一个问题。

## 6. 知识卡片视觉设计系统

本节是所有定位卡、行动卡及后续系列卡片的唯一视觉规范。只替换文案和数据，不再临时发明颜色、组件或版式。

### 6.1 视觉目标与画幅

当用户要求图片、海报或可分享卡片，且当前 Agent 具备网页、SVG、Canvas 或图片渲染能力时，把 Markdown 卡片分别渲染成独立图片。

- 风格：**极简知识海报 / Editorial Infographic / Insight Card**。
- 推荐画幅：**1080 × 1620 px**；信息较少时可采用竖版 3:4。
- 卡片背景固定为米白，不使用照片、纹理、渐变或装饰性插画。
- 页面使用现代无衬线字体：`PingFang SC`、`Microsoft YaHei`、`Helvetica Neue`、Arial 依次回退。
- 内容区推荐上、下内边距 72px，左、右内边距 76px。
- 留白承担主要分层功能；分割线只在标题区使用一次，模块之间靠间距分层。

### 6.2 固定色板与颜色语义

颜色不是装饰，同一种颜色在所有卡片里必须表达同一种角色。

| 颜色 | 色值 | 固定用途 |
|---|---:|---|
| 墨黑 Ink | `#1A1A1A` | 正文、描边、唯一强调条底色 |
| 柠青 Lime | `#D7E85C` | 核心结论、CTA、唯一高亮色 |
| 暖粉 Pink | `#F4B4C6` | 人、情绪、关系类信息 |
| 杏橙 Orange | `#F0A868` | 风险、待调整、中间状态 |
| 雾紫 Purple | `#BEB0E8` | 方法、结构、系统类信息 |
| 米白 Cream | `#F6F2E8` | 全部卡片的固定背景 |
| 柠青浅底 Lime Tint | `#F0F4D0` | 浅底信息框、Summary 区 |
| 灰棕 Gray | `#8A8574` | 英文眉题、辅助文字、注释 |

需要浅色状态框时，使用与语义色对应的低饱和浅底：暖粉浅底 `#FBE4EA`、杏橙浅底 `#FBE3C9`、雾紫浅底 `#EAE4F8`。四色出现顺序可以变化，但语义不得互换。

### 6.3 字体层级

同一屏最多保留三级主要字号：主标题、模块标题、正文。英文眉题和注释共同使用 15px，不形成新的视觉层级。

| 层级 | 规范 |
|---|---|
| 英文眉题 / 注释 | 15px，700，灰棕色；眉题字距 4px、全大写 |
| H1 主标题 | 50px，800，行高 1.15，字距 -0.5px |
| 模块标题 | 26px，800 |
| 正文 | 19px，行高 1.75 |

- 一句话尽量不超过两行。
- 不通过缩小字号塞入更多内容。
- 注释只补充背景，不承担主要结论。

### 6.4 间距体系

- 模块间距：56–60px。
- 模块标题到正文：22–26px。
- 正文行高：1.7–1.8 倍。
- 卡片内边距：72px / 76px（纵向 / 横向）。
- 徽章与正文左边缘采用 60px 对齐：模块说明和正文相对徽章左侧缩进 60px。
- 主标题下方可使用 3px 墨黑分割线；正文模块之间不得反复增加通栏线。

### 6.5 固定组件库

后续卡片只能从以下组件中选择，不新增相似但不一致的样式。

1. **数字徽章 Number Badge**：44 × 44px，10px 圆角，粗体数字；颜色按模块语义选择，用于形成清晰的编号分区。仅用于真正有顺序或分类意义的模块。
2. **信息卡 Day / Item Card**：白底、2px 墨黑描边、14px 圆角、约 20px × 22px 内边距；内部可放彩色胶囊标签和一个核心问题或结论。
3. **强调条 Action Bar**：墨黑底、米白文字、14px 圆角；标签使用柠青色。一张卡片只允许一处黑底反白，用于全卡唯一需要“喊出来”的行动。
4. **三栏浅色块 Tri Box**：浅语义色底、2px 墨黑描边、14px 圆角；用于并列的三类信号或三种状态，每格文案控制在两行以内。
5. **总结条 Summary Bar**：柠青浅底 `#F0F4D0`、顶部 3px 墨黑线、左侧 14px 柠青圆点并带 2px 墨黑描边；只写一句核心结论。

### 6.6 强制使用规则

1. 每张卡最多 5 个模块；超过就合并强相关信息或拆成下一张卡，不得硬塞。
2. 默认仍交付“内容定位卡 + 7 天验证卡”两张图片。若 Markdown 字段超过 5 个视觉模块，先合并相关字段；只有无法保证可读性时才增加图片，并向用户说明。
3. 颜色语义固定：柠青 = 核心结论；暖粉 = 人 / 情绪 / 关系；杏橙 = 风险 / 待调整；雾紫 = 方法 / 结构 / 系统。
4. 一张卡只允许一处黑底反白强调条。
5. 数字徽章只用于真实顺序或分类，不为了装饰而编号。
6. 正文单段不超过 2 行；信息过多时拆成信息卡，不写长段落。
7. 不为了视觉效果删除置信度、证据状态、最大待验证假设或商业连接。

### 6.7 两张卡片的模块映射

**内容定位卡**最多使用以下 5 个视觉模块：

1. 一句话定位；
2. 优先人群 + 唯一核心主线；
3. 为什么是你；
4. 内容结构（核心线 / 证明线 / 人格与关系线）；
5. 商业连接；“暂时不要做”和判断状态可合并进本模块或 Summary，不另起视觉模块。

**7 天定位验证卡**最多使用以下 5 个视觉模块：

1. 本轮验证目标；
2. 7 天行动（两列信息卡，DAY 7 使用唯一黑色强调条）；
3. 三类观察信号；
4. 保留 / 调整 / 放弃或降级；
5. 下一轮迭代。

### 6.8 禁止事项

- 不使用复杂插画、大量图标、emoji、照片、渐变彩虹和装饰性背景。
- 不制作传统商业 PPT、复杂信息图或花哨营销海报。
- 不增加第四级主要字号、第二处黑色强调条或新的组件类型。
- 不复制参考图中的具体文案、水印、账号信息或品牌标识。

## 7. 图片生成与渲染规则

Markdown 是所有 Agent 的必交付。图片是有渲染能力时的增强交付，不得因为无法生成图片而省略卡片内容。

文本密集型卡片优先使用 HTML/CSS、SVG 或 Canvas，确保中文、色值、字号、边距和组件尺寸准确；不要依赖容易写错中文或随意改变版式的自由生成式图片模型。

渲染时使用以下规范：

```text
生成 1080 × 1620 px 的极简编辑式中文知识卡片。
固定米白背景 #F6F2E8，墨黑文字与描边 #1A1A1A。
使用 PingFang SC / Microsoft YaHei 无衬线字体，英文眉题 15px、宽字距、灰棕色 #8A8574；
中文主标题 50px/800，模块标题 26px/800，正文 19px/1.75。
内容区纵向内边距 72px、横向内边距 76px，模块间距 56–60px。
使用 44×44px、10px 圆角的数字徽章；信息卡为白底、2px 黑边、14px 圆角。
柠青 #D7E85C 只表示核心结论与 CTA；暖粉 #F4B4C6 表示人和关系；
杏橙 #F0A868 表示风险或待调整；雾紫 #BEB0E8 表示方法和系统。
每张卡最多 5 个模块，只允许一处墨黑底反白的 Action Bar。
底部 Summary 使用 #F0F4D0 浅底、3px 黑色上边框和带黑边的柠青圆点。
不要人物、插画、复杂图标、emoji、照片、渐变或营销海报效果。
```

渲染后必须检查：中文是否准确、色值是否正确、字号层级是否超出三级、模块是否超过 5 个、是否出现第二处黑色强调条，以及换行、裁切和留白是否正常。每张卡片独立成图。

## 8. 输出前检查

- [ ] 是否只有一个核心定位和一类优先人群？
- [ ] 是否提供真实信任证据，并区分事实、推断和假设？
- [ ] 证明线和人格与关系线是否都服务于核心主线？
- [ ] 是否包含轻量商业连接和明确取舍？
- [ ] 7 天计划是否在验证定位，而非单纯日更？
- [ ] 是否包含目标用户、商业行动和个人能量信号？
- [ ] 是否提供保留、调整和放弃或降级规则？
- [ ] Markdown 是否避免大段分析并适合截图？
- [ ] 图片版是否使用圆角编号、白色信息框、宽松留白和底部浅色总结区？
- [ ] 是否在 10 秒内可以识别核心定位或核心行动？

任何一项不满足，先压缩文案或调整结构，再输出。
