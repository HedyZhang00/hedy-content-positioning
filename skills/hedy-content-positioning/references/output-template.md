# 知识卡片输出模板

## 输出原则

- 用户确认候选后，默认只输出两张 Markdown 知识卡片。
- 结论优先，每个模块 1 至 4 行；不用长篇散文解释。
- 如果用户明确要求极简，可合并成一张卡片，但不得删去置信度、最大待验证假设和验证决策。
- 卡片文字应便于直接截图，或交给设计工具渲染成米白底、黑色粗标题、清晰分割线、局部高亮和底部总结条的视觉卡片。

## 候选确认前

先输出 2 至 3 个候选，每个候选使用：

```markdown
### 候选 N｜名称

- 服务谁：
- 核心问题：
- 为什么是你：
- 商业连接：
- 持续素材：
- 主要风险：
- 待验证假设：
- 五维判断：证据强度【强/中/弱】｜商业连接【强/中/弱】｜长期能量【强/中/弱】｜差异化【强/中/弱】｜验证成本【低/中/高】
```

候选后只问：你想选择、组合还是否决哪个候选？

## 卡片 1｜内容定位卡

```markdown
# 一人公司内容定位卡
CONTENT POSITIONING CARD

## 01｜一句话定位
我帮助【优先人群】，解决【具体阶段的核心问题】，主要分享【方法、经验或视角】，让她们能够【期望改变】。

## 02｜优先目标人群
- 【只保留一类人及其阶段】

## 03｜唯一核心主线
【未来半年希望被记住的中心问题】

## 04｜为什么是你
- 【真实案例、结果、求助、经历或数据，最多 3 条】

## 05｜内容结构
- 核心主线：【中心问题】
- 证明线：【案例、方法、结果、失败或数据】
- 人格与关系线：【价值观、经历、选择或相关观察】

## 06｜商业连接假设
- 连接业务：【业务方向】
- 可能付费的人：【目标人群】
- 可能付费的问题：【具体问题】
- 最小产品或服务：【轻量假设】
- 内容后的下一步：【评论、私信、问卷或预约中的一个】
- 已有商业证据：【事实；没有则写暂无】
- 最大待验证假设：【一句话】

## 07｜判断状态
- 当前置信度：【高/中/待验证】
- 已知事实：【1 至 3 条】
- 合理推断：【1 至 2 条】

> SUMMARY｜这是当前证据下最值得先验证的“内容定位假设 V1.0”，不是终身结论。
```

## 卡片 2｜7 天验证行动卡

```markdown
# 7 天验证行动卡
7-DAY VALIDATION CARD

## 01｜首要任务
【这 7 天只验证的一个核心定位假设】

## 02｜7 天行动
- DAY 1：【目标人群问题】｜验证：【假设】｜引导：【动作】
- DAY 2：【资格与经历】｜验证：【假设】｜引导：【动作】
- DAY 3：【典型场景】｜验证：【假设】｜引导：【动作】
- DAY 4：【方法或框架】｜验证：【假设】｜引导：【动作】
- DAY 5：【案例或复盘】｜验证：【假设】｜引导：【动作】
- DAY 6：【观点或反常识】｜验证：【假设】｜引导：【动作】
- DAY 7：【需求验证】｜验证：【假设】｜引导：【私信/问卷/预约】

## 03｜观察信号
- 目标用户信号：【要记录什么】
- 商业行动信号：【要记录什么】
- 个人能量信号：【要记录什么】

## 04｜7 天后决策
- 保留：目标用户明确回应，且你愿意继续表达。
- 调整：有流量但人群不对，或有共鸣但没有具体问题与行动。
- 放弃或降级：没有目标用户回应，也缺少素材和继续表达的意愿。

> SUMMARY｜定位不是想出来的，而是用真实发布、反馈和行动信号验证出来的。
```

## 禁止事项

- 不输出一万字报告或重复推理过程。
- 不把三条内容线写成三个平级定位。
- 不跳过商业连接假设。
- 不只给七个选题，而不写每条在验证什么。
- 不用固定点赞量替代目标用户、商业行动和个人能量判断。
