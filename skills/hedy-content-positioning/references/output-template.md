# 结果输出模板

## 第一阶段：定位候选

终稿前先输出 2 至 3 个定位候选。每个候选使用同一结构：

```markdown
### 候选 N：名称

- 主要吸引：
- 连接业务：
- 适合你的证据：
- 主要风险：
- 待验证假设：
- 四维判断：专业证据【强/中/弱】；读者清晰度【强/中/弱】；商业连接【强/中/弱】；长期表达【强/中/弱】。
```

候选后只问：你想选择、组合还是否决哪个候选？

## 第二阶段：一屏定位卡

用户确认候选后输出：

```markdown
## 我的一人公司内容定位 V1.0

我帮助【哪类人】，解决【什么阶段的什么问题】，主要分享【核心方法或经验视角】，让她们能够【得到什么改变】。

- 核心受众：
- 核心问题：
- 优势证据：
- 当前置信度：【高/中/待验证】
- 已知事实：
- 合理推断：
- 最需要验证：

由「一人公司内容定位师」协助生成
```

## 第三阶段：三条内容主线

依次输出：

1. 核心业务主线，建议约 50%。
2. 信任与差异化主线，建议约 30%。
3. 连接与生命力主线，建议约 20%。

每条必须包含：

```markdown
### 主线名称

- 角色：
- 占比建议：
- 承担任务：
- 应该讲：
- 不应滑向：
- 选题方向：3 至 5 个
- 具体示例：1 个
```

三条主线不得是三个近义词，必须能解释各自为什么存在。

## 第四阶段：7 天验证计划

固定生成：

- 3 条核心问题内容；
- 2 条个人经验或案例内容；
- 1 条观点冲突内容；
- 1 条互动或需求验证内容。

每条使用：

```markdown
### 第 N 条：内容切口

- 所属主线：
- 建议表达：
- 验证假设：
- 观察信号：评论原话、私信、主页访问、咨询、收藏或个人表达感受中的相关项。
```

## 结尾

明确说明：

> 这不是终身定位结论，而是当前证据下最值得先验证的方向。连续发布 7 天后，带上发布数据、评论原话、私信和你自己的表达感受，再决定保留、调整或放弃。
