## 1. 输出目标与原则

最终结果用于帮助用户作出定位与行动决策，不输出长篇分析报告。

默认输出两张独立知识卡片：

1. **内容定位卡**：回答“我服务谁、讲什么、为什么是我、内容连接什么业务”；
2. **7 天定位验证卡**：回答“接下来怎么测试、观察什么、如何作出取舍”。

最终结论统一称为“内容定位假设 V1.0”，不得写成命定方向、唯一答案或确定性人格结论。

### 表达规则

- 结论优先，每个模块控制在 1 至 4 行。
- 只保留决策需要的信息：人群、问题、证据、取舍、商业连接和行动。
- 显式区分已知事实、合理推断和待验证假设。
- 不编造用户没有提供的数据、案例、反馈或付费需求。
- 两张卡片总字数建议控制在 700 至 1200 字；图片版应进一步压缩。
- 除非用户明确要求，不在卡片后继续输出额外长篇解释。

## 2. 定位候选

用户确认候选前，先输出 2 至 3 个方向：

```markdown
### 候选 N｜名称

- 服务谁：
- 核心问题：
- 为什么是你：
- 商业连接：
- 持续素材：
- 主要风险：
- 待验证假设：
- 五维判断：证据强度【强/中/弱】｜商业连接【强/中/弱】｜长期能量【强/中/弱】｜差异化【强/中/弱】｜验证成本【低/中/高】
```

候选后只问：你想选择、组合还是否决哪个候选？用户确认前不得生成最终卡片。

## 3. 卡片一：内容定位卡

```markdown
CONTENT POSITIONING

# 一人公司内容定位卡
> 内容定位假设 V1.0

## 01｜你的定位
我帮助【一类优先人群】，解决【具体阶段的核心问题】，通过【独特方法或能力】，让她们能够【期望改变】。

## 02｜你服务谁
【具体身份】正处于【具体阶段】，最需要解决【具体问题】。

## 03｜唯一核心主线
围绕「【未来半年最值得反复讨论的中心问题】」持续输出。

你希望别人记住你是因为：【一句清晰记忆点】。

## 04｜为什么是你
- 【真实案例、结果或可验证经历 1】
- 【真实案例、结果或可验证经历 2】
- 【真实案例、结果或可验证经历 3】

## 05｜内容结构
- 核心主线：【中心问题】
- 证明线：【案例、方法、结果、失败复盘或数据】
- 人格与关系线：【价值观、经历、选择或相关观察】

证明线和人格与关系线都服务于核心主线，不是三个独立定位。

## 06｜商业连接假设
- 业务方向：【内容最可能连接的业务】
- 可能付费的人：【最可能产生付费行为的人群】
- 付费问题：【对方愿意花钱解决的具体问题】
- 最小产品或服务：【当前能力范围内可测试的轻量方案】
- 下一步行动：【私信、问卷、预约或社群中的一个】
- 已有商业证据：【事实；没有则写暂无直接证据】
- 最大待验证假设：【一句话】

## 07｜暂时不要做
- 不要同时讲【容易分散定位的方向】。
- 不要为了流量持续吸引【非目标人群】。
- 不要在证据不足时包装成【尚未成立的身份】。

## 08｜判断状态
- 当前置信度：【高/中/待验证】
- 已知事实：【1 至 3 条】
- 合理推断：【1 至 2 条】

> 总结 / SUMMARY
> 你真正需要放大的，不是【过去容易陷入的方向】，而是【最值得建立的核心认知】。未来半年，优先围绕「【核心主线】」积累内容、案例和反馈。
```

一句话定位控制在 35 至 60 字。只选择一类优先人群和一个内容中心。“为什么是你”只保留最强的三条真实证据。证据不足时明确写成公开实践型或成长型定位。

商业连接只搭桥，不展开完整产品矩阵、详细定价、成交话术、私域 SOP 或高客单交付方案。

## 4. 卡片二：7 天定位验证卡

```markdown
7-DAY VALIDATION

# 7 天定位验证卡
> 不继续猜定位，用真实内容验证定位。

## 01｜本轮要验证
【目标人群】是否会因为你讨论「【核心问题】」而产生共鸣、信任或进一步行动。

## 02｜7 天行动
- DAY 1｜身份与问题：【内容主题】｜验证：【问题定义】｜引导：【留言】
- DAY 2｜典型痛点：【内容主题】｜验证：【用户是否说这就是我】｜引导：【补充场景】
- DAY 3｜核心观点：【内容主题】｜验证：【观点记忆点】｜引导：【同意或反对】
- DAY 4｜方法框架：【内容主题】｜验证：【收藏、尝试或追问】｜引导：【实践反馈】
- DAY 5｜案例证据：【内容主题】｜验证：【信任是否增强】｜引导：【私信问题】
- DAY 6｜人格与选择：【内容主题】｜验证：【价值观认同】｜引导：【分享选择】
- DAY 7｜商业行动：【内容主题】｜验证：【是否愿意进入下一步】｜引导：【私信/问卷/预约】

## 03｜重点观察
- 目标用户信号：【具体评论、问题、自我识别和新增关注】
- 商业行动信号：【私信、问卷、预约、服务或合作询问】
- 个人能量信号：【最容易持续和最明显消耗的主题】

## 04｜7 天后判断
- 保留：目标用户明确回应，出现具体问题或下一步行动，同时你愿意继续表达。
- 调整：有流量但人群不对，或有共鸣却没有具体问题与商业行动。
- 放弃或降级：多次发布后没有目标用户回应，也缺少素材、意愿或业务连接。

## 05｜下一轮迭代
带回表现最好内容、评论原话、私信咨询、新增目标用户和个人能量记录，再生成“内容定位假设 V1.1”。

> 总结 / SUMMARY
> 定位不是一次想清楚的，而是在持续表达中逐步收敛的。7 天的目标不是证明你一定正确，而是减少一个错误方向。
```

7 天计划不能只是机械日更。每条内容都必须标明验证目的和行动引导。不得只用点赞量判断定位。

## 5. 信息不足时的输出

六项最小证据门槛缺少两项以上时，不得强行填满两张卡片。只输出：

```markdown
# 定位信息缺口卡

- 已确认事实：
- 当前无法确定：
- 缺失的关键证据：
- 下一步只补充的一项信息：

> 目前还不足以形成可靠的内容定位假设。不是因为你没有定位，而是缺少【具体证据】。
```

然后继续一次只问一个问题。

## 6. 知识卡片视觉与版式规范

### 6.1 视觉目标

当用户要求图片、海报或可分享卡片，且当前 Agent 具备网页、SVG、Canvas 或图片渲染能力时，把两张 Markdown 卡片分别渲染成独立图片。

整体风格：**极简知识海报 / Editorial Infographic / Insight Card**。

核心视觉：米白背景、黑色粗标题、01/02/03 编号分区、黑色细横线、少量低饱和荧光色块、充足留白和底部高亮总结区。不得做成传统商业 PPT、复杂信息图或花哨营销海报。

### 6.2 卡片数量与画幅

- 默认两张独立卡片，不把所有信息塞入一张图。
- 画幅采用竖版 3:4，推荐尺寸 **1200 × 1600 px**。
- 四周保留明显留白，正文不得贴边。
- 使用米白、浅奶油或暖灰白背景，不使用满版照片和复杂纹理。

### 6.3 标题系统

- 顶部先放小号全大写英文眉题：`CONTENT POSITIONING` 或 `7-DAY VALIDATION`。
- 中文主标题使用黑色粗体大字号、左对齐，是全卡最醒目的元素。
- 标题区与正文使用黑色细横线分隔。
- 中文采用现代、易读的无衬线字体；不得混用过多字体。

### 6.4 模块与色彩

- 正文使用 01、02、03 等编号分区，每个模块只表达一个问题。
- 模块之间使用横向细线；编号可置于小色块中。
- 以黑色文字和米白背景为主，单张卡片最多使用三种强调色。
- 推荐荧光黄绿色、柔和粉色、浅橙色或淡紫色。
- 色彩只用于编号、重点词、对比模块和底部总结，不大面积堆叠高饱和颜色。

### 6.5 文字密度

- 模块标题不超过 12 个字；每个模块 2 至 5 行。
- 单条要点尽量控制在 25 字以内；图片版正文建议不超过 500 字。
- 内容过多时，依次删除重复解释、理论背景、修饰语言和次要方向。
- 始终保留结论、证据、取舍、商业连接和行动。
- 不缩小字号来硬塞文字；必要时进一步压缩文案，而不是破坏留白。

### 6.6 对比与总结区

只有存在明确对比价值时，才使用左右双栏，例如“核心主线 / 暂时不要做”或“保留 / 调整 / 放弃”。每栏只放 2 至 4 个短要点。

每张卡片底部必须设置独立高亮区：

```text
总结 / SUMMARY
```

总结使用一块荧光黄绿色或其他单一强调色，内部保留充足留白，只写 1 至 2 句有记忆点的最终判断。

### 6.7 两张卡片的视觉中心

- 内容定位卡：突出“一句话定位 + 唯一核心主线”；其他模块都用于支撑这两个结论。
- 7 天定位验证卡：突出“验证什么 + 如何判断”；七个选题不得挤占全卡大部分空间。

### 6.8 禁止事项

- 不使用复杂插画、大量图标、emoji、渐变彩虹和装饰性背景。
- 不制作营销海报式口号，不使用空泛鼓励。
- 不复制参考图的具体文字、水印、账号信息或品牌标识。
- 不为了视觉效果删除置信度、证据状态或最大待验证假设。

## 7. 图片生成与渲染规则

Markdown 是所有 Agent 的必交付。图片是有渲染能力时的增强交付，不得因为无法生成图片而省略卡片内容。

文本密集型卡片优先使用 HTML/CSS、SVG 或 Canvas 确保中文准确；不要依赖容易写错中文的自由生成式图片模型。

图片渲染提示可使用：

```text
极简编辑设计风格的中文知识卡片，竖版 3:4，1200 × 1600 px。
米白色纸张质感背景，黑色粗体无衬线大标题，顶部小号全大写英文眉题。
正文使用 01、02、03 编号分区，模块之间用黑色细横线分隔。
局部使用荧光黄绿色、柔和粉色和浅橙色色块强调重点。
信息清晰克制，留白充足，无人物、无插画、无复杂图标、无 emoji。
底部有醒目的“总结 / SUMMARY”高亮色块。
整体接近高质量杂志编辑信息图和小红书知识卡片，不是 PPT，不是营销海报。
```

渲染后检查中文文字、数字、换行、裁切和留白。每张卡片独立成图。

## 8. 输出前检查

- [ ] 是否只有一个核心定位和一类优先人群？
- [ ] 是否提供真实信任证据，并区分事实、推断和假设？
- [ ] 证明线和人格与关系线是否都服务于核心主线？
- [ ] 是否包含轻量商业连接和明确取舍？
- [ ] 7 天计划是否在验证定位，而非单纯日更？
- [ ] 是否包含目标用户、商业行动和个人能量信号？
- [ ] 是否提供保留、调整和放弃或降级规则？
- [ ] Markdown 是否避免大段分析并适合截图？
- [ ] 图片版是否为竖版 3:4、编号分区、少量强调色和底部总结区？
- [ ] 是否在 10 秒内可以识别核心定位或核心行动？

任何一项不满足，先压缩文案或调整结构，再输出。
